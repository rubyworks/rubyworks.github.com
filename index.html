  • XMLToolKit  [Download]  [API Documentation]  [xml:Proof Documentation]  [xml:Proof Specification]  [xml:Tails Specification]

    Collection XML tools. I've now gathered all my XML work into a single library set. These tools are at various stages of completion. This library includes an assortment of tools:


    • PrettyXML - PrettyXML is a pretty printer for XML/XHTML. It has definable indentation and makes for a good demonstration of REXML's stream parser, which is required.


    • REREXML - REREXML adds a few extra features to REXML, like absolute xpaths, both relative and indexed, and a parser for Namespace and Schema processing instructions (non-standard).


    • XMLProof/Ruby - An implementation of xml:Proof, an alternate XML schema language. xml:Proof has more features and is easier to use than any other schema language out there. Have your doubts? Can your schema language validate content based on its alphabetical order? Can your schema language require two attributes and three elements be present if any one of them is, even though they appear is vastly different sections of the document? Can your schema language validate content against Regular Expressions? These are just a few of the things xml:Proof can do.


    • XMLTailor/Ruby - An implementation of xml:Tails, an alternate XML stylesheet transformation language. xml:Tails is a tag attribute language for scripting languages. This implementation is obviously for use with Ruby, but implementations could be made for other lanuages. It it perfect for Web Developers who need to keep a distinct seperation between WYSIWYG Web design and underlying scripting logic. It also makes a good substitue for ERuby, and other web templating systems. Sorry, XMLTailor/Ruby is not yet available as it is alpha code and exceedingly slow, although is does work. Hopefully I can speed it up enough to be of reasonable use some time soon.


    • SQLiX - SQLiX is descendent of XSQL, which you may have read about in the Ruby's Developers Guide. SQLiX makes adavances over XSQL by allowing referenced sub-records to be imported into the generated XML very easily. It also includes XiSQL, providing the reverse process of updating, inserting, and deleting properly formed XML documents against the database. (Note: This is an outcropping of Jigsaw, which turned out to be infeasible.)

  • GUtopIa  [Download]  [API Documentation]

    NOTE: GUtopIa is undergoing a massive overhaul, so who knows how much of the below applies.
    GUtopIa is a GUI meta-API for the Ruby scripting language. We say "meta" because it presents a single common API to the developer regardless of the underlying base API. In sum, GUtopIa intends to provide a best-of-all-worlds GUI toolkit to the Ruby community of programmers, meeting all requirements of such a toolkit: cross-platform compatability, native platform integration, ease of coding with good Seperation Of Concerns and opertaing on the Principle of Least Suprise, international font support, as well as a good dose of flexibity and speed. There are two major divisions to the GUtopIa meta-API, tentatively nicked name Biz and Joy. Biz uses wxWindows as a backend GUI API, while Joy uses the ClanLib Game SDK. Both are open source projects in their own right. The common meta-API as presented to the developers is desgined around a Dual Channel Model-View-Control architecture. This structure provides optimal SOC. The core-application (Model) need not be necessarily altered in anyway in order to give it a user-interface (View). The two are coorelated by pairs of intermediary conduits, which are actually dynamically generated methods in the ruby source. These shuttle state changes "automagically" between core-application and user-interface (Controller). The GUtopIa project is an offspring of the original RougeProject, a part of FreeRIDE and continues in its footsteps with the goal of bringing GUI programming to new heights that only a languge like Ruby can provide.
